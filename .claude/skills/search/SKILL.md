# /search — 联网搜索调研

对指定主题进行多源联网搜索，汇总对比后生成 Word 文档。

## 使用方式
```
/search <主题描述>
```
示例：
- `/search 2026年最好的AI生图API对比`
- `/search RTX 5060 Ti vs 4060 Ti 性能对比`
- `/search Python异步任务队列框架对比`

## 流程
1. 根据用户主题，拆分为 3-5 个搜索角度（如：功能、价格、性能、生态、国内可用性）
2. 对每个角度用 WebSearch 搜索，每个角度至少搜 2 次（不同关键词）
3. 对搜索结果中的关键页面用 WebFetch 抓取详情验证
4. 汇总所有数据，整理成对比表格
5. 用 python-docx 生成 Word 文档，保存到 `reports/` 目录

## 输出规范
- 文档包含：概述、对比表格、推荐方案、数据来源列表
- 每条数据标注来源（URL 或平台名）
- 不确定的数据标注"待验证"，禁止伪造
- 文件命名：`reports/<主题关键词>_调研报告.docx`

## 规则
- 搜索前先告知用户搜索计划（几个角度、关键词），等用户确认
- 中文主题优先搜中文源，英文主题优先搜英文源
- 价格数据必须来自官网或可信电商平台，不能凭印象编造
- 搜索完成后先在对话中展示摘要，用户确认后再生成 Word
