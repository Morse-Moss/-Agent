# 单入口部署与跨电脑迁移说明

## 一、目标

本说明用于支持以下两种场景：

- 将当前项目迁移到另一台 Windows 电脑继续运行
- 以单入口方式启动系统，让用户通过一个地址访问页面和 API

## 二、当前技术状态

- 当前默认数据库：`SQLite`
- 数据库文件：`backend/data/app.db`
- 图片与资产目录：`backend/storage/`
- 前端部署产物目录：`frontend/dist/`
- 数据库切换入口：`APP_DATABASE_URL`

当前还没有正式切换到 `MySQL`，但配置入口已经保留。

## 三、迁移到另一台电脑需要带走什么

至少需要保留以下内容：

| 项目 | 是否必须 | 说明 |
| --- | --- | --- |
| 项目代码目录 | 是 | 包含前后端代码与脚本 |
| `backend/data/app.db` | 是 | 当前业务数据库 |
| `backend/storage/` | 是 | 上传图、生成图和导出图 |
| `docs/source/` | 否 | 如果需要继续维护文档源稿则建议保留 |

## 四、另一台电脑需要准备什么

- Python 3.11 或更高版本
- Node.js 18 或更高版本

## 五、开发模式启动

前后端分开运行：

```bat
start-backend.cmd
start-frontend.cmd
```

访问地址：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:8000`

## 六、单入口部署模式启动

使用：

```bat
start-app.cmd
```

该脚本会先构建前端，再启动后端，由后端统一提供前端页面和 `/api` 接口。

访问地址：

```text
http://127.0.0.1:8000
```

## 七、环境变量说明

可参考：

```text
backend/.env.example
```

部署建议至少配置以下变量：

| 变量名 | 用途 |
| --- | --- |
| `APP_SECRET_KEY` | 服务端签名密钥 |
| `APP_DEFAULT_ADMIN_USERNAME` | 默认管理员账号 |
| `APP_DEFAULT_ADMIN_PASSWORD` | 默认管理员密码 |
| `APP_DATABASE_URL` | 数据库连接串 |
| `APP_SERVE_FRONTEND` | 是否由后端托管前端构建产物 |

## 八、MySQL 准备状态

当前系统仍默认使用 `SQLite`，但后续可以切换到 `MySQL`。如果要启用 MySQL：

1. 安装 `backend/requirements-mysql.txt`
2. 将 `APP_DATABASE_URL` 修改为 MySQL 连接串
3. 重启后端服务

示例连接串：

```text
mysql+pymysql://username:password@127.0.0.1:3306/ecom_art_agent?charset=utf8mb4
```

## 九、云端公开访问还缺什么

如果后续要让用户通过公网网址登录和使用，目前还缺：

- 正式域名
- HTTPS 证书
- 反向代理
- 进程托管
- 正式数据库
- 文件备份或对象存储
- 更严格的安全配置

## 十、验证方式

后端启动后，可执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check-single-entry.ps1
```

验证项包括：

- 根路径是否返回前端页面
- `/create` 这类前端路由刷新后是否仍能打开
- `/health` 是否正常返回
