# 单入口部署与跨电脑迁移说明

## 一、目标

本说明用于支持以下两种场景：

- 将当前项目迁移到另一台 Windows 电脑继续运行
- 以单入口方式启动系统，让用户通过一个地址访问页面和 API

## 二、当前技术状态

- 当前默认数据库：`MySQL`
- 当前 MySQL 安装目录：`G:\MySQL`
- 图片与资产目录：`backend/storage/`
- 前端部署产物目录：`frontend/dist/`
- 数据库切换入口：`APP_DATABASE_URL`

当前项目已经正式切换到 `MySQL`，`SQLite` 仅作为历史迁移源保留。

## 三、迁移到另一台电脑需要带走什么

至少需要保留以下内容：

| 项目 | 是否必须 | 说明 |
| --- | --- | --- |
| 项目代码目录 | 是 | 包含前后端代码与脚本 |
| `G:\MySQL\` 数据目录或导出的数据库备份 | 是 | 当前业务数据库 |
| `backend/storage/` | 是 | 上传图、生成图和导出图 |
| `docs/internal/source/` | 否 | 如果需要继续维护文档源稿则建议保留 |
| `docs/deliverables/` | 否 | 如果需要带着现成 Word 交付件一起展示则建议保留 |

## 四、另一台电脑需要准备什么

- Python 3.11 或更高版本
- Node.js 18 或更高版本
- 如果目标电脑没有 MySQL，可直接复用项目内的安装脚本和本地安装包

## 五、开发模式启动

前后端分开运行：

```bat
start-backend.cmd
start-frontend.cmd
```

访问地址：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:8000`

## 六、单入口部署模式启动

使用：

```bat
start-app.cmd
```

该脚本会先构建前端，再启动后端，由后端统一提供前端页面和 `/api` 接口。

访问地址：

```text
http://127.0.0.1:8000
```

## 七、环境变量说明

可参考：

```text
backend/.env.example
```

部署建议至少配置以下变量：

| 变量名 | 用途 |
| --- | --- |
| `APP_SECRET_KEY` | 服务端签名密钥 |
| `APP_DEFAULT_ADMIN_USERNAME` | 默认管理员账号 |
| `APP_DEFAULT_ADMIN_PASSWORD` | 默认管理员密码 |
| `APP_DATABASE_URL` | 数据库连接串 |
| `APP_SERVE_FRONTEND` | 是否由后端托管前端构建产物 |

## 八、MySQL 迁移与恢复

当前系统已默认使用 MySQL，建议目标电脑按以下顺序恢复：

1. 使用 `scripts/ops/install_mysql_gdrive.ps1` 安装或初始化 MySQL
2. 使用 `scripts/ops/restore_mysql_and_storage.py` 恢复数据库备份和图片资产
3. 检查 `backend/.env` 中的 `APP_DATABASE_URL`
4. 运行 `start-app.cmd` 或 `start-backend.cmd`

示例连接串：

```text
mysql+pymysql://username:password@127.0.0.1:3306/ecom_art_agent?charset=utf8mb4
```

## 九、云端公开访问还缺什么

如果后续要让用户通过公网网址登录和使用，目前还缺：

- 正式域名
- HTTPS 证书
- 反向代理
- 进程托管
- 正式数据库
- 文件备份或对象存储
- 更严格的安全配置

## 十、验证方式

后端启动后，可执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev\check-single-entry.ps1
```

验证项包括：

- 根路径是否返回前端页面
- `/create` 这类前端路由刷新后是否仍能打开
- `/health` 是否正常返回
