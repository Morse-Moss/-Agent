# 文档导航

这个目录保留的是适合放在 GitHub 仓库中的公开文档，优先服务于“看懂项目、跑起项目、接入生图能力、准备部署”这几类场景。

## 建议保留并上传的文档

- [使用说明](使用说明.md)
  - 适合快速了解系统怎么使用、页面怎么操作。
- [接入生图 API 说明](接入生图API说明.md)
  - 适合配置外部图片 Provider。
- [部署与迁移说明](部署与迁移说明.md)
  - 适合本地迁移、单入口运行和后续云端部署准备。
- [接口示例](api-examples.http)
  - 适合本地调试和接口验证。

## 建议本地保留、不上传 GitHub 的内容

- 根目录下的 `.docx` 成品文档
- `docs/source/` 中的内部方案源稿
- 阶段性过程记录，例如 `docs/当前进度.md`
- 本地调试截图、临时文件、锁文件

## 原则

GitHub 仓库优先保留以下内容：

- 能帮助别人快速理解项目的说明
- 能帮助别人运行项目的指南
- 能帮助别人部署项目的资料

以下内容默认不作为公开仓库主内容：

- 内部版本总结
- 过程性复盘
- 仅用于本地交付的 Word 成品
- 临时调试产物
